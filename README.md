# 📋 Attendance Report Processor

> **Automatically extract, process, and regenerate employee attendance reports from PDF files — with OCR support for Hebrew documents.**

---

## 🚀 Overview

**Attendance Report Processor** is a Python CLI tool that reads Israeli attendance report PDFs, extracts structured data using OCR (Tesseract), applies business logic (time variation rules), and generates a clean, formatted PDF output.

It supports **two distinct report formats** (Type A and Type B), auto-detected from the document content, and handles Hebrew text natively.

---

## ✨ Features

- 📄 **PDF OCR Extraction** — Converts scanned PDF attendance sheets to structured data using Tesseract + pdf2image
- 🔍 **Auto Report-Type Detection** — Classifies between Report Type A (simple hourly) and Type B (125%/150% overtime, breaks)
- 🔢 **Structured Parsing** — Extracts dates, entry/exit times, hours, payments, totals, and overtime columns
- ⚙️ **Time Variation Engine** — Applies configurable business rules to shift/adjust working hours
- 📑 **PDF Report Generation** — Produces a clean, formatted PDF output using ReportLab / WeasyPrint
- 🐳 **Docker Ready** — Full containerized support for consistent cross-platform execution
- 🇮🇱 **Hebrew Language Support** — Tesseract OCR configured with `tesseract-ocr-heb` for Hebrew text

## 📦 Requirements

### Local Setup

- Python **3.11+**
- [Tesseract OCR](https://github.com/tesseract-ocr/tesseract) with **Hebrew** language pack
- Poppler utilities (`poppler-utils`)

Install system dependencies (Ubuntu/Debian):

```bash
sudo apt-get install -y tesseract-ocr tesseract-ocr-heb poppler-utils libgl1 libglib2.0-0
```

Install Python dependencies (using [uv](https://github.com/astral-sh/uv)):

```bash
uv sync
```

Or with pip:

```bash
pip install .
```

## ▶️ Usage

### CLI

```bash
python main.py <path/to/report.pdf> [-o <output_directory>]
```

**Arguments:**

| Argument | Description | Default |
|---|---|---|
| `input` | Path to the input attendance PDF | *(required)* |
| `-o` / `--output` | Directory to save the generated PDF | `.` (current dir) |

**Example:**

```bash
python main.py ./reports/attendance_april.pdf -o ./output
```

## 🐳 Docker

### Build & Run with Docker Compose

Edit `docker-compose.yml` to set your input/output paths:

```yaml
volumes:
  - "/path/to/your/input_pdfs:/data/input"
  - "./output:/app/output"
command: ["/data/input/your_report.pdf", "-o", "/app/output"]
```

Then run:

```bash
docker compose up --build
```

### Manual Docker Run

```bash
# Build the image
docker build -t attendance-report .

# Run with mounted volumes
docker run --rm \
  -v "/path/to/input:/data/input" \
  -v "$(pwd)/output:/app/output" \
  attendance-report /data/input/report.pdf -o /app/output
```

## 📐 Report Types

### Type A — Standard Hourly Report
- Fields: date, day, entry time, exit time, total hours, hourly rate, total payment
- Typical keywords: `מחיר לשעה`, `לתשלום`

### Type B — Overtime / Break Report
- Fields: date, day, entry time, exit time, break time, 100%/125%/150% overtime columns, Saturday hours
- Typical keywords: `הנשר`, `הפסקה`, `125`, `150`

## ⚠️ Error Handling

The system uses a structured exception hierarchy rooted in `AttendanceBaseException`:

| Exception | When raised |
|---|---|
| `OCRProcessingError` | PDF not found or OCR fails |
| `UnknownReportTypeError` | Report type cannot be determined |
| `ParsingError` | Row/report parsing fails |
| `TransformationError` | Variation/time rule violation |
| `PDFGenerationError` | PDF rendering fails |
| `ValidationError` | Business rule violation (e.g. exit < entry) |
| `TimeBoundaryError` | Time outside allowed range |
| `BreakValidationError` | Invalid break duration |
